사용법 기본

Mizutama Conte는 애니메이션 제작용으로, 콘티를 그려서 애니매틱(영상으로 재생할 수 있는 콘티)으로 재생할 수 있는 에디터입니다. 데이터는 모두 사용자 PC 안에서만 처리되며 서버로 전송되지 않습니다. 데스크톱 버전(Mac / Windows / Linux)과 브라우저에서 동작하는 Web 버전을 제공합니다.

이 페이지에서는 프로젝트를 만들고 내보내기까지의 전체 흐름을 순서대로 설명합니다. 키보드 단축키 목록은 별도 페이지(단축키)에 정리해 두었습니다.

전체 흐름

  1. 프로젝트를 만든다 (해상도·화면비를 고른다)
  2. 컷의 그림을 그린다 (외부 페인트 앱으로 PSD 파일을 편집)
  3. 컷을 편집한다 (컷 추가·씬 구분 편집·컷 결합과 분리·레이어 추가, ACTION / DIALOGUE / TIME 편집)
  4. 카메라 워크를 넣는다 (크롭과 리사이즈, 슬라이더 편집)
  5. 듀레이션을 정한다 (스톱워치로 실연하며 측정)
  6. 트랜지션을 넣는다 (페이드·크로스페이드)
  7. 프리뷰로 재생해서 확인한다
  8. PDF나 MP4로 내보낸다

작업 중간의 저장은 자동으로 이루어집니다(PSD와 JSON). 명시적인 저장 조작은 기본적으로 필요 없습니다.

1. 프로젝트를 만든다

새 프로젝트

메뉴에서 "새 프로젝트"를 선택합니다. 생성 시점에, 앞으로 작업할 콘티의 해상도와 화면비를 정합니다.

해상도는 SD / HD / FHD / 2K / 4K 중에서, 화면비는 4:3 / 16:9 / 1.85:1 / 2.39:1 중에서 고릅니다. 여기서 정한 해상도가 PSD의 기준 크기(네이티브 해상도)가 됩니다.

고민된다면, TV 애니메이션이라면 HD 또는 FHD의 16:9, 극장용 작품을 상정한다면 1.85:1 또는 2.39:1 정도가 기준이 됩니다.

각본(Markdown)에서 만든다

작성한 각본이 이미 Markdown으로 되어 있다면, 거기서 한 번에 씬과 컷을 만들 수 있습니다. 메뉴의 "각본에서 새로 만들기"(Cmd / Ctrl + Shift + N)에서 불러옵니다. 제목(헤딩)이 씬의 구분, 수평선이 컷의 구분, 대사 행이 DIALOGUE가 되며, 각 컷에 빈 PSD가 자동으로 만들어집니다.

표기법의 자세한 내용과 구체적인 예시는 각본(Markdown) 작성법을 참조하세요.

2. 그림을 그린다

Mizutama Conte 자체에는 드로잉 도구가 없습니다. 그림은 평소 사용하는 페인트 앱으로 그립니다. CLIP STUDIO PAINT / Photoshop / Affinity Photo / GIMP / Krita 등, PSD 파일을 읽어 들여 덮어쓰기 저장할 수 있는 것이라면 어떤 소프트웨어든 사용할 수 있습니다.

PSD를 열어서 그린다

  1. 컷의 칸(PICTURE)을 더블클릭합니다.
  2. 설정에서 선택한 페인트 앱이 실행되고, 그 컷의 실체인 .psd 파일이 열립니다.
  3. 그대로 그린 뒤 덮어쓰기 저장합니다.
  4. Mizutama Conte로 돌아오면, 저장된 내용이 자동으로 읽혀 칸의 표시가 갱신됩니다.

독자적인 포맷으로 변환하지 않습니다. 열리는 것은 일반 PSD이며, 늘 쓰던 소프트웨어에서 그대로 편집할 수 있습니다.

3. 컷을 편집한다

콘티는 씬과 컷(CUT)의 모음입니다. 목록의 각 행이 하나의 컷에 해당합니다.

컷을 추가한다

행을 추가하면 새로운 빈 컷이 들어갑니다. 각 컷에는 빈 PSD가 만들어지므로, 그대로 그림을 그리기 시작할 수 있습니다.

씬을 구분한다

원하는 위치에서 씬을 구분할 수 있습니다. 씬 구분을 넣으면, 거기서부터 새로운 씬이 시작됩니다.

컷을 결합·분리한다

인접한 컷은 결합할 수 있습니다. 반대로, 결합한 컷은 분리해서 원래대로 되돌릴 수 있습니다. 컷을 다시 나눌 때 사용합니다.

레이어＝컷 안의 칸

하나의 컷은 여러 레이어를 가질 수 있습니다. 여기서 말하는 레이어는, 컷 안의 시계열 플립북(움직이는 그림책)으로 다뤄집니다. 배경 위에 겹친 레이어가 한 칸씩 순서대로 표시됩니다. 인비트윈이나 움직임이 있는 컷을, 한 컷 안에서 표현하기 위한 구조입니다. 여러 레이어를 묶어서 한 칸으로 다루고 싶은 경우에는 레이어를 그룹화합니다.

ACTION / DIALOGUE / TIME

각 컷에는 다음 정보를 적어 넣을 수 있습니다.

  • ACTION(지문): 컷의 내용이나 움직임에 대한 설명
  • DIALOGUE(대사): 그 컷에서 말하는 대사
  • TIME(듀레이션): 그 컷의 길이

ACTION과 DIALOGUE는 그 자리에서 텍스트를 편집할 수 있습니다. TIME은 숫자로 입력하는 것 외에, 뒤에서 설명할 스톱워치로도 정할 수 있습니다.

4. 카메라 워크를 넣는다

컷 위에서 카메라 프레임(크롭 프레임)을 드래그하여 크기와 위치를 정합니다. 프레임을 크게 하면 줌인, 위치를 옮기면 팬을 표현할 수 있습니다.

드래그 중에 Shift를 누르면 스냅이 적용됩니다.

  • 가로 방향 스냅 (수평 팬)
  • 세로 방향 스냅 (수직 팬)
  • 비율을 유지한 스냅 (화면비를 유지한 채로 하는 줌)

리사이즈 방향에서 기본 카메라 워크를 자동 생성

카메라 프레임을 리사이즈하면, 그 방향(어느 쪽으로 넓혔는지 / 줄였는지)으로부터 그럴듯한 팬이나 줌의 초깃값이 자동으로 들어갑니다. 우선 대략적으로 프레임을 움직여 감을 잡고, 나중에 세밀하게 조정하는 흐름이 편합니다.

슬라이더와 숫자로 조정한다

카메라 워크 패널에서는, 슬라이더와 숫자 입력 양쪽으로 값을 다듬을 수 있습니다.

  • 슬라이더로 대략 맞추고, 숫자로 정확하게 정한다
  • IN과 OUT은 서로 바꿀(스왑) 수 있다
  • 네이티브 해상도 가득(=잘라내기 없음, 움직임 없음) 상태인 항목은 회색으로 비활성화된다

회색 비활성화는 "여기는 카메라 워크가 걸려 있지 않다"는 표시입니다.

5. 듀레이션을 정한다 (스톱워치)

컷의 길이는, 실제로 연기를 해 보고 그 시간을 그대로 듀레이션으로 입력할 수 있습니다.

  1. 왼쪽 툴바에서 T(스톱워치) 도구를 선택합니다.
  2. 듀레이션을 재고 싶은 컷을 선택합니다.
  3. Space로 측정을 시작하고, 다시 한 번 Space로 정지합니다.
  4. 연기한 길이가 그 컷의 TIME(초 + 프레임)으로 들어갑니다.

대사를 소리 내어 읽거나, 움직임을 몸으로 재현하는 등, 머릿속의 타이밍을 그대로 숫자로 만들 수 있습니다. 손으로 초 수를 입력하고 싶은 경우에는 TIME을 직접 편집해도 됩니다.

6. 트랜지션을 넣는다

컷이 바뀌는 부분에 페이드와 크로스페이드를 설정할 수 있습니다.

  • 페이드인·페이드아웃: White / Black 두 종류를 고를 수 있습니다.
  • 크로스페이드: 앞 컷과 다음 컷이 겹쳐서 바뀌는 페이드입니다.

트랜지션 듀레이션의 기본값은 0.5초입니다(예: 24fps라면 12프레임, 30fps라면 15프레임). 길이는 나중에 조정할 수 있습니다.

7. 프리뷰로 재생한다

프리뷰를 재생하면, 설정한 듀레이션·카메라 워크·트랜지션이 모두 반영된 상태로, 애니매틱으로서 처음부터 끝까지 볼 수 있습니다. 팬이나 줌은 실제 움직임으로 재생되므로, 타이밍이나 화면의 흐름을 그림을 그린 단계에서 확인할 수 있습니다.

8. 내보낸다

완성된 콘티는 두 가지 형식으로 내보낼 수 있습니다.

PDF 인쇄(콘티 용지)

스튜디오의 콘티 용지에 가까운 서식(A4)으로 내보냅니다. 각 컷의 PICTURE에는 IN / OUT 카메라 프레임이 표시되고, 누적 듀레이션도 들어갑니다. 종이 운용에 그대로 올릴 수 있습니다.

MP4 동영상(애니매틱)

프로젝트의 네이티브 해상도·fps로 MP4 형식의 동영상 파일로 내보냅니다. 프리뷰에서 확인한 움직임이 그대로 동영상이 됩니다.

지원 환경

데스크톱 버전의 지원 환경은 다음과 같습니다.

  • macOS 12 이상
  • Windows 10 이상
  • Linux(Ubuntu 20.04 상당 이상)

이보다 오래된 환경에서는, Chromium 계열 브라우저에서 동작하는 Web 버전을 이용하세요.

Web 버전은 Chromium 계열 브라우저(Chrome / Edge 등)에서의 이용을 상정하고 있습니다. 파일의 저장·읽기나 동영상 내보내기는 File System Access API·WebCodecs에 의존하므로, Safari나 Firefox에서는 열람만 가능합니다(저장·내보내기 불가). PDF 인쇄도 Chromium 계열을 권장합니다.

배포하는 빌드는 현재 코드 서명을 하지 않았기 때문에, 첫 실행 시 OS의 경고를 우회하는 한 가지 절차가 필요합니다(Mac이라면 Gatekeeper 허용 등).

그 밖에

  • 자동 저장: 편집 내용은 PSD와 JSON에 수시로 저장됩니다.
  • 최근 연 프로젝트: 지난번까지의 작업을 바로 다시 열 수 있습니다.
  • Undo / Redo: 조작의 취소와 다시 실행을 할 수 있습니다.
  • 표시 언어: 한국어 / 일본어 / 영어를 전환할 수 있습니다.
  • 테마: 라이트 / 다크를 전환할 수 있습니다.

조작 단축키는 단축키 페이지에 정리해 두었습니다.
