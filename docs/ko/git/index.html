Git 연동

데스크톱 버전에는 프로젝트를 git으로 버전 관리하는 기능이 있습니다. 원하는 시점에 그 순간의 상태를 기록해 두고, 나중에 이전 상태로 되돌리거나 변경 이력을 따라가 볼 수 있습니다. Web 버전에는 이 기능이 없습니다.

자동 저장은 항상 동작하며, 편집할 때마다 그 내용이 파일에 기록됩니다. git 연동은 그 위에서 "여기까지의 상태"를 구분해 남겨 두기 위한 구조입니다.

시작하기

프로젝트를 연 상태에서 git 패널의 "버전 관리 시작"을 누르면, 그 프로젝트 폴더가 git의 관리 아래로 들어갑니다. 이때 다음 작업이 자동으로 이루어집니다.

  • 폴더를 git 저장소로 초기화합니다
  • .gitignore와 .gitattributes를 기록합니다
  • 이 저장소에 대해 git-lfs를 활성화합니다
  • 첫 커밋을 만듭니다

이미 git으로 관리하고 있는 프로젝트를 연 경우에는, 그대로 이어서 사용할 수 있습니다.

스냅샷을 남긴다

변경 사항이 있을 때 "스냅샷 만들기"를 누르면, 그 시점의 상태가 커밋으로 기록됩니다. 메시지를 적어 남길 수도 있고, 비워 두면 날짜·시간을 사용한 이름으로 기록됩니다.

변경 사항이 없을 때는 버튼을 누를 수 없습니다. 마지막으로 언제 기록했는지도 패널에서 확인할 수 있습니다.

무엇이 기록되고, 무엇이 제외되는가

기록되는(추적되는) 것은 콘티의 내용 그 자체입니다.

  • .json: 프로젝트의 구성이나 설정
  • .psd: 각 컷의 그림(뒤에서 설명하듯이 git-lfs로 관리)

한편, 다음 항목은 .gitignore로 제외됩니다.

.DS_Store
*.pdf
*.mp4
*.mov
.trash/

PDF나 동영상(MP4 / MOV)은 콘티에서 내보낸 결과물입니다. 원본 데이터에서 몇 번이든 다시 만들 수 있으므로 이력에는 포함하지 않습니다. .trash/는 앱 안에서 삭제한 것을 두는 곳이고, .DS_Store는 macOS가 만드는 관리용 파일이므로, 둘 다 기록하지 않습니다.

큰 PSD는 git-lfs로 다룬다

.psd는 이미지라서 파일 크기가 커지기 쉽습니다. 이것을 그대로 git에 넣으면 저장소가 빠르게 부풀어 오르고, 이력을 다루기도 무거워집니다. 그래서 .psd는 git-lfs(Git Large File Storage)로 관리합니다.

버전 관리를 시작하면 .gitattributes에 다음 한 줄이 기록됩니다.

*.psd filter=lfs diff=lfs merge=lfs -text

이렇게 하면 저장소 본체에는 가벼운 포인터만 들어가고, PSD의 실체는 lfs 쪽에 저장됩니다. 평소 작업에서 git-lfs를 의식할 필요는 없습니다.

필요한 것

git 연동을 사용하려면 git과 git-lfs가 모두 설치되어 있어야 합니다. 둘 중 하나라도 찾을 수 없을 때는, git 패널에 그 사실과 설치 안내가 표시됩니다. 설치한 뒤 앱을 다시 시작하면 기능을 사용할 수 있습니다.
