Git 連携

デスクトップ版には、プロジェクトを git でバージョン管理する機能があります。任意のタイミングでその時点の状態を記録し、あとから前の状態に戻したり、変更の履歴をたどったりできます。Web 版にはこの機能はありません。

自動保存は常に動いていて、編集はその都度ファイルに書き込まれます。git 連携は、その上で「ここまでの状態」を区切って残しておくための仕組みです。

始め方

プロジェクトを開いた状態で、git のパネルから「バージョン管理を開始」を押すと、そのプロジェクトフォルダが git の管理下に入ります。このとき次のことが自動で行われます。

  • フォルダを git リポジトリとして初期化する
  • .gitignore と .gitattributes を書き込む
  • git-lfs をこのリポジトリに対して有効にする
  • 最初のコミットを作る

すでに git で管理しているプロジェクトを開いた場合は、そのまま続きから使えます。

スナップショットを残す

変更があるときに「スナップショットを作成」を押すと、その時点の状態がコミットとして記録されます。メッセージを書いて残すこともできますし、空のままでも日時を使った名前で記録されます。

変更がないときはボタンは押せません。最後にいつ記録したかも、パネルで確認できます。

何が記録され、何が除外されるか

記録される（追跡される）のは、絵コンテの中身そのものです。

  • .json: プロジェクトの構成や設定
  • .psd: 各カットの絵（後述のとおり git-lfs で管理）

一方、次のものは .gitignore で除外されます。

.DS_Store
*.pdf
*.mp4
*.mov
.trash/

PDF や動画（MP4 / MOV）は、絵コンテから書き出した結果です。元データから何度でも作り直せるので、履歴には含めません。.trash/ はアプリ内で削除したものの置き場、.DS_Store は macOS が作る管理用ファイルなので、どちらも記録しません。

大きな PSD は git-lfs で扱う

.psd は画像なので、ファイルサイズが大きくなりがちです。これをそのまま git に入れると、リポジトリが急速に膨らみ、履歴の扱いも重くなります。そこで .psd は git-lfs（Git Large File Storage）で管理します。

バージョン管理を開始すると、.gitattributes に次の 1 行が書き込まれます。

*.psd filter=lfs diff=lfs merge=lfs -text

これにより、リポジトリ本体には軽いポインタだけが入り、PSD の実体は lfs 側に保存されます。普段の操作で git-lfs を意識する必要はありません。

必要なもの

git 連携を使うには、git と git-lfs の両方がインストールされている必要があります。どちらかが見つからないときは、git のパネルにその旨と導入の案内が表示されます。インストールしてからアプリを再起動すると、機能が使えるようになります。
