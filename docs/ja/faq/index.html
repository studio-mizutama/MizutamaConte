よくある質問

Mizutama Conte について、よく寄せられる質問をまとめました。

Mizutama Conte とはどんなアプリですか？

アニメ制作の絵コンテを作って管理するためのエディタです。カットの絵を描き、セリフ・デュレーション・カメラワークを付けて、1 本の絵コンテにまとめます。

パンやズームを設定すると、その動きが付いたまま映像としてプレビュー再生できます。仕上げた絵コンテは、制作現場で使われる書式に近い PDF や、実際に動く MP4 として書き出せます。Mac / Windows / Linux のデスクトップ版と、ブラウザで動く Web 版があり、データはすべて手元のマシンで完結します。

始めるには何をすればいいですか？

まずブラウザで Web 版を開いて試すのが手軽です。インストールは要りません。普段づかいするなら、自動保存や外部ペイントソフト連携が使えるデスクトップ版をおすすめします。

入手方法は ダウンロード、プロジェクトの作り方から書き出しまでの流れは 使い方の基本 のページにまとめています。

AI 連携機能はありますか？

いいえ。Mizutama Conte に、AI による生成や補助の機能はありません。絵を描くのも、構成を決めるのも、カメラワークを付けるのも、すべて人の手で行います。連携するペイントソフトに AI 機能がある場合、それを使用することは可能です。Mizutama Conte には AI 機能を入れる予定はありません。

プレビュー専用のアプリですか？

いいえ。Mizutama Conte は、絵コンテを作って管理するためのエディタです。カットを並べ、セリフ・デュレーション・カメラワークを編集し、PDF や動画に書き出すところまでを担当します。

カメラワークを付けたまま映像としてプレビュー再生できるのは特徴のひとつですが、数ある機能の一つで、プレビューだけのツールではありません。

なぜアプリ内に描画機能がないのですか？

描く作業は、ふだん使っているペイントソフトのほうが速くて快適だからです。カットをダブルクリックすると、お使いのペイントソフトで本物の .psd が開きます。描いて保存すれば、自動で読み込み直されます。

簡易的なペイント機能を実装することは技術的には可能ですが、別タブ・別ウインドウで編集することになるので、シームレスな体験の観点からも、内部にペイント機能を持たせるメリットは大きくないと考えています。中途半端なお絵描き機能を抱えるより、使い慣れた道具をそのまま活かせるほうがいいと考えています。

対応しているペイントソフトは何ですか？

CLIP STUDIO PAINT、Photoshop、Affinity Photo、GIMP、Krita など、PSDファイルを読み込んで、上書き保存できるソフトウェアであれば、どのソフトウェアでもお使いいただけます。

Web 版とデスクトップ版に機能の違いはありますか？

基本の編集機能は同じですが、デスクトップ版だけの機能がいくつかあります。

  • カットをダブルクリックしてペイントソフトで開く機能と、保存後に自動で読み込み直す機能は、デスクトップ版だけです。Web 版では、Finder やエクスプローラーからローカルのファイルを直接開いて描き、描き終えたら自分で読み込み直します。
  • git 連携（プロジェクトのバージョン管理）も、デスクトップ版だけの機能です。

普段づかいにはデスクトップ版が快適です。インストールせずにすぐ試したいときや、対応 OS の都合でデスクトップ版が動かないときは Web 版を使ってください。

開発は継続していますか？

はい。現在も開発を続けています。機能追加や不具合の修正は、GitHub のリポジトリ で進めていて、更新履歴やリリースもそこで公開しています。

サーバーにデータは送信されますか？

いいえ。描いた絵コンテのデータが外部のサーバーに送られることはありません。Mizutama Conte は手元のマシンの中だけで動きます。アカウント登録もログインもなく、利用状況を集める仕組み（テレメトリ）も入れていません。

デスクトップ版はローカルのファイルとして、Web 版はブラウザのファイルシステム API を通じて、自分のディスク上のデータを直接読み書きします。

クラウド同期や共同編集はできますか？

Mizutama Conte 自体には、クラウド同期や複数人での同時編集の機能はありません。データは手元のマシンの中だけにあります。

複数の環境で同じプロジェクトを扱いたいときは、Dropbox などクラウドストレージのフォルダにプロジェクトを置くか、デスクトップ版の git 連携を使ってください。

自動保存をオフにすることはできますか？

いいえ、自動保存をオフにする設定はありません。編集すると、その都度ファイルに保存されます。

過去の状態を残しておきたいときは、次のどちらかをおすすめします。ひとつは、Dropbox などスナップショット（バージョン履歴）を自動で残してくれるクラウドストレージの中にプロジェクトを置くこと。もうひとつは、デスクトップ版の git 連携でバージョン管理することです。どちらも、ある時点の状態に戻せるようになります。

git 連携とは何ですか？

デスクトップ版には、プロジェクトを git でバージョン管理する機能があります。新しいプロジェクトを git 管理で始めることも、すでにあるプロジェクトを後から git 管理下に置くこともできます。あとは好きなタイミングでコミットすれば、その時点の状態を記録として残せます。前の状態に戻したり、変更の履歴をたどったりできます。

何が記録され何が除外されるか、大きな PSD をどう扱うか（git-lfs）といった詳しい話は、Git 連携 のページにまとめています。

対応しているブラウザは何ですか？

Web 版は、Chromium 系のブラウザ（Google Chrome、Microsoft Edge など）での利用を想定しています。保存や書き出しに File System Access API と WebCodecs を使うためです。

Safari と Firefox では、これらの API が揃っていないため、閲覧はできても保存・書き出しはできません。制作するときは Chrome か Edge、またはデスクトップ版を使ってください。

Web 版で Documents フォルダ直下のフォルダを読み取れません

ブラウザのセキュリティ上の制限です。File System Access API は、Documents・Desktop・Downloads といった特別なフォルダと、その直下のフォルダへのアクセスをブロックします。ブラウザ側の仕様なので、アプリからは変えられません。

プロジェクトのフォルダを、これらの直下ではなく一段深い場所に置くと読み込めます。たとえば Documents の中に作業用フォルダをひとつ作り、プロジェクトはそのさらに中に置く、という形です。別のドライブや任意の作業フォルダを使うのも有効です。

どの OS に対応していますか？

デスクトップ版は macOS 12 以降、Windows 10 以降、Linux（Ubuntu 20.04 相当以降）で動きます。これより古い環境では、Chrome か Edge で Web 版を使ってください。対応環境の詳細は ダウンロード のページにあります。

どんな形式で保存されますか？

絵は .psd、構成や設定の情報は .json で保存します。どちらも一般的な形式なので、中身を他のツールで開いて確認できます。独自フォーマットに閉じ込めることはしません。

PDF や動画で書き出せますか？

はい。制作現場で使われる書式に近い絵コンテ用紙の PDF（A4）と、ネイティブ解像度・fps の MP4 を書き出せます。MP4 はカメラワークが付いたまま、プレビューで見たとおりに出力されます。書き出しの手順は 使い方の基本 のページで説明しています。

脚本を読み込んで絵コンテの雛形を作ることはできますか？

はい。Markdown形式で書かれた脚本のインポートに対応しています。詳細な記法は 脚本インポート機能 のページで説明しています。

動画や音声の読み込みはできますか？

いいえ。現在のところ、動画や音声のインポート機能はありません。音声とのタイミング合わせは、ストップウォッチ機能の使用をおすすめします。今後のアップデートで機能の追加は検討しています。

不具合を見つけました。要望はどこに送ればいいですか？

GitHub のリポジトリの Issues で受け付けています。再現する手順や、お使いの OS・ブラウザを書き添えてもらえると助かります。

商用ライセンスなど込み入った相談は、ランディングページの問い合わせ先からメールでも構いません。
