# 脚本インポート記法

脚本を Markdown で書いておくと、Mizutama Conte がそれを読み込んで、シーンと CUT に切り分けた絵コンテの下地を一気に作ります。セリフ（DIALOGUE）とト書き（ACTION）も振り分けて、各 CUT には空の PSD が用意されます。あとは絵を描き、尺やカメラワークを詰めていく、という流れを想定しています。

このページでは、読み込まれる Markdown の書き方を、入力例と生成結果の対応つきで説明します。

## 起動の仕方

メニューの「脚本から新規」から実行します。ショートカットは Cmd+Shift+N（Windows / Linux は Ctrl+Shift+N）です。

1. 脚本の Markdown ファイルを選びます。拡張子は `.md` のほか、`.txt` も受け付けます。
2. 通常の新規作成（Cmd+N / Ctrl+N）と同じように、解像度・アスペクト比・fps を設定します。
3. 読み込むと、脚本の中身からシーンと CUT が組まれた新しいプロジェクトができます。

書式が想定と違っていても、アプリが落ちることはありません。読み取れない場合はエラーダイアログで知らせます。

## 全体の構造

脚本 Markdown は、おおまかに次の階層で解釈されます。

- 先頭の h1（`# 見出し`）= 作品タイトル（＝プロジェクト名）
- 見出し（`##` 以下、レベルは問わない）= シーンの区切り。見出しのテキストがシーン名
- 水平線（`---` など）= CUT の区切り
- 「名前：セリフ」「名前「セリフ」」の行 = DIALOGUE
- それ以外の本文 = ACTION

順番に見ていきます。

## h1 は作品タイトル

ファイルの先頭にある h1 が、作品タイトルとしてプロジェクト名に使われます。

```
# 雨上がりの放課後
```

h1 のすぐ下に書いた説明文（次の見出しが来るまで）は、読み込みの対象になりません。あらすじやメモを置いておく場所として使えます。

先頭に h1 がなく、いきなり h2 から始まっている場合は、Markdown のファイル名がプロジェクト名になり、その h2 以降はシーン区切りとして読まれます。先頭に見出しがまったくない場合は、最初の見出しが現れるまでの内容は読み飛ばされます。

## 見出しはシーンの区切り

h2 から h6 までの見出しは、どれもシーンの区切りとして同じ意味を持ちます。階層をそろえる必要はありません。`##` でも `####` でも構いません。見出しのテキストがそのシーンのタイトルになります。

途中に出てくる h1 も、ここでは h2 以下と同じくシーン区切りとして扱います。

シーンタイトルは、そのシーンの最初の CUT に付きます。空のシーン（中身のない見出し）が作られることはありません。

## 水平線は CUT の区切り

`---` のような水平線で CUT を区切ります。書き方の方言にはある程度対応していて、次のものはすべて CUT 区切りとして扱われます。

```
---
***
___
- - -
```

記号が 3 つ以上だけの行であれば水平線とみなします。

## ACTION と DIALOGUE の振り分け

CUT の中身は、行ごとに DIALOGUE（セリフ）か ACTION（ト書き）に振り分けられます。

DIALOGUE と判定されるのは、次の 2 つの形です。

- `名前：セリフ` のように、行頭にキャラクター名とコロンが続く形。コロンは半角 `:` でも全角 `：` でも構いません。コロンの後ろのスペースは省略できます。
- `名前「セリフ」` のように、名前のあとにカギ括弧で囲んだ形。

名前部分は 20 文字までを目安にしています。それ以外の本文はすべて ACTION になります。

入力例:

```
ハル：おはよう、今日も暑いね
ナギ:コロンの後ろにスペースが無くても大丈夫
ナギ「カギ括弧でも同じく DIALOGUE」
教室。窓の外でセミが鳴いている。
```

この 4 行が 1 つの CUT に入ると、次のように振り分けられます。

- DIALOGUE: 「おはよう、今日も暑いね」「コロンの後ろにスペースが無くても大丈夫」「カギ括弧でも同じく DIALOGUE」
- ACTION: 「教室。窓の外でセミが鳴いている。」

セリフの名前（`ハル：` や `ナギ「`）は、書いたとおりそのまま残ります。プレフィックスを勝手に削ったりはしません。

## Markdown 記法は外して中身だけ取り込む

引用・コードブロック・箇条書きも、中身は ACTION として取り込みます（セリフの形になっていなければ）。読み込みのときに Markdown の記号は外して、中身のテキストだけを残します。

たとえば次の入力は、

```
> 古い校舎の廊下
- 黒板に残った落書き
* 窓際の机
**強い西日**が差し込む
```

ACTION として、こう取り込まれます。

```
古い校舎の廊下
黒板に残った落書き
窓際の机
強い西日が差し込む
```

`-` や `>`、`**...**`（太字）、`*...*`（斜体）、`` `...` ``（インラインコード）などの記号は外れ、中身だけが残ります。改行はそのまま保ちます。1 つの CUT の中に複数行あれば、改行で結合して 1 つの ACTION にまとめます。

## 空の CUT は作らない

区切りと区切りのあいだに、ACTION も DIALOGUE も無ければ、その CUT は作りません。`---` を続けて書いてしまっても、間に中身がなければ無視されます。

各 CUT には、ACTION か DIALOGUE の少なくともどちらかに必ず文字が入る、という形になります。

シーン見出しの直前に CUT 区切りの `---` を入れても入れなくても、結果は同じです。最終 CUT のあとに区切りが残っていても、適切に処理します。

## まとまった例

次の脚本を読み込んだとします。

```
# 雨上がりの放課後

あらすじをここに書いておける（読み込みでは無視される）

## 教室

ハル：やっと雨やんだね
窓を開けると、濡れたグラウンドの匂い。

---

ナギ「先生、もう帰っていいですか」
時計は午後 5 時を指している。

## 昇降口

ハル「靴、湿ってる」
```

読み込むと、次のようになります。

- 作品タイトル: 雨上がりの放課後
- シーン「教室」
  - CUT 1 — DIALOGUE「やっと雨やんだね」 / ACTION「窓を開けると、濡れたグラウンドの匂い。」
  - CUT 2 — DIALOGUE「先生、もう帰っていいですか」 / ACTION「時計は午後 5 時を指している。」
- シーン「昇降口」
  - CUT 3 — DIALOGUE「靴、湿ってる」 / ACTION なし

シーンタイトルは、それぞれのシーンの最初の CUT（CUT 1 と CUT 3）に付きます。h1 直下のあらすじは読み込まれません。

## 読み込んだあと

各 CUT には空の PSD（`c001.psd`、`c002.psd`、…）が自動で用意されます。PICTURE をダブルクリックすれば外部のお絵描きアプリで開いて描き始められます。CUT の尺は仮の値が入るので、ストップウォッチ（T ツール）や TIME の直接編集で実際の尺に合わせていってください。

セリフとト書きが入った状態から始められるので、絵に集中しやすくなるはずです。

## 注意点

- 拡張子は `.md` のほか `.txt` も受け付けますが、中身は Markdown として解釈します。
- 1 行に書いた `名前：` や `名前「` は DIALOGUE になります。ト書きの中で偶然この形になっている行は、セリフとして拾われることがあります。
- 想定外の書式でも読み込み自体は止まりません。結果が思ったものと違うときは、見出しや `---` の位置、セリフの書き方を見直してください。
