# 脚本mdインポート機能

- このmdは脚本インポート機能の仕様書兼、インポートされるmdのテンプレートを兼ねてます
- 最初のh1は作品タイトル。つまり、インポート時はプロジェクト名として使われます
- h1直下の文章（次のh以外の見出しがくるまで）は、インポート時無視されます
- h2〜h6の見出しは、シーンの区切りを表します
- h2、h3、h4、、、どれを使おうと同じ意味を持ちます。階層化する必要はないです
- もし冒頭にh1がなく、いきなりh2から始まる場合は、mdのファイル名をプロジェクト名としてインポートして、h2以降はシーン区切りとして解釈したら良いです
- もし冒頭にいかなる見出し要素もなければ、最初に見出し要素が来るまで内容はインポートの対象外とします


インポートのやり方

- ファイル→脚本から新規（cmd or ctrl + shift + N）
- インポートする脚本mdを選ぶ
- 形式がmdであれば、拡張子はtxtも受け付けます
- 解像度やfps、アスペクト比等、新規（cmd or ctrl + N）で設定できる項目は「脚本から新規」でも設定できるべきです

モンガー:コロンの後にスペース忘れてても同様
むい：このように、コロンが全角でも同様です
モンガー「このように、キャラ名＋カッコで囲まれた文も同様です」

---

mdの区切り線は、カットの区切りを表します。
このように、セリフとして解釈できない行は、そのカットのACTIONに入れます
むい: mdの方言にも柔軟に対応するべきです

***

モンガー「区切り線は`---`の他に、`***`や`___`など、いろんな書き方に対応したいね」

___

> このように引用形式のものや、

```txt
このようにコード形式のものや
```

- このような箇条書きや

* 箇条書きの別の表現など

すべてACTIONに入れるべきものです

ただし、mdの記法はインポート時は読み取らなくて良いです
- あああ
なら、
あああ
とインポートされるべきです
**太字**や*斜体*等の記号もインポートせずに、中身だけインポートされるべきです
改行の有無もインポートの対象です
